    Create `src/sanity/components/hungarianHolidays.ts` exporting a function `getHungarianHolidays(year: number): Array<{date: string; name: string}>`.

    Fixed Hungarian national holidays (these do NOT move):
    - January 1 — Újév (New Year)
    - March 15 — Nemzeti ünnep (1848 Revolution)
    - May 1 — A munka ünnepe (Labour Day)
    - August 20 — Az államalapítás ünnepe (St. Stephen's Day)
    - October 23 — Az 1956-os forradalom ünnepe (1956 Revolution)
    - November 1 — Mindenszentek (All Saints' Day)
    - December 25 — Karácsony első napja (Christmas Day)
    - December 26 — Karácsony második napja (St. Stephen's Day / Boxing Day)

    Moveable holidays (Easter-based):
    - Nagypéntek (Good Friday) — Easter Sunday minus 2 days
    - Húsvéthétfő (Easter Monday) — Easter Sunday plus 1 day
    - Pünkösdhétfő (Whit Monday) — Easter Sunday plus 50 days

    For Easter calculation, implement the Anonymous Gregorian algorithm (Computus) — a pure function that takes year and returns the Easter Sunday date. This is a well-known algorithm, no external library needed.

    Return dates as ISO strings (YYYY-MM-DD). Export the function and also export the type.

    **Custom Sanity input component:**
    Create `src/sanity/components/BlockedDatesInput.tsx` — a custom Sanity form input component for the `dates` array field on blockedDate.

    This is a Sanity custom input component. It receives standard Sanity input props (value, onChange, etc.) via the `InputProps` type from `sanity`. Use `useCallback` for event handlers and `set`/`unset` patch operations from `sanity`.

    The component renders:
    1. A **month navigation** header with left/right arrows and "YYYY. hónap" display (Hungarian month names)
    2. A **calendar grid** showing days of the current month
    3. A **"Magyar ünnepnapok hozzáadása"** button that pre-populates Hungarian holidays for the displayed year (only adds holidays not already in the array)
    4. A **legend** showing color coding: normal day, blocked day, holiday (blocked + isHoliday)

    **Interaction (locked decisions from CONTEXT.md):**
    - Single click: selects the START of a range (visual highlight)
    - Shift+click: selects the END, blocks ALL dates between start and end (inclusive)
    - If a date is already blocked, clicking it unblocks it (removes from array)
    - No individual date clicking for adding — always range-based. Single click sets start, shift-click sets end. If user clicks without shift after setting start (without shift-clicking end), it resets the start to the new click.
    - Holidays can be unblocked by clicking them (per CONTEXT.md: "Doctor can unblock pre-populated holidays if needed")

    **Mobile (CONTEXT.md says "press-and-slide gesture"):**
    - On touch devices, use onTouchStart to set range start and onTouchMove to update range end as finger slides across dates. onTouchEnd commits the range.
    - Visual preview of the range while sliding (highlighted but not yet committed)

    **Styling:**
    Use Sanity UI components where possible (Card, Button, Text, Flex, Grid from `@sanity/ui`). For the calendar grid, use a CSS grid with 7 columns. Color blocked dates with a muted red/pink background, holidays with an orange/amber background. Selected range preview in light blue.

    **Wiring to schema:**
    Update `blockedDateType.ts` — on the `dates` array field, add `components: { input: BlockedDatesInput }` to wire the custom input. Import BlockedDatesInput at the top of the file.

    **Important implementation notes:**
    - The value is an array of `{_key: string, date: string, isHoliday: boolean}` objects
    - Use `set` patch to update the entire array when ranges are added/removed
    - Generate `_key` using a deterministic approach (e.g., the date string itself or `nanoid` from Sanity)
    - The component must handle the case where value is undefined (empty array — first time opening)
    - Hungarian month names: ["január", "február", "március", "április", "május", "június", "július", "augusztus", "szeptember", "október", "november", "december"]
    - Hungarian day abbreviations for column headers: ["H", "K", "Sze", "Cs", "P", "Szo", "V"]
