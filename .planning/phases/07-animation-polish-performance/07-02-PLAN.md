**Create `src/components/motion/CircleWipeLink.tsx`** — a 'use client' component.

**Purpose:** A drop-in replacement for next/link that plays a circle wipe animation before navigating to internal content pages.

**Interface:**
```tsx
interface CircleWipeLinkProps {
  href: string;
  children: React.ReactNode;
  className?: string;
  // Pass through any other props to the underlying element
}
```

**Trigger rules (from CONTEXT.md):**
- PLAYS circle wipe for: internal content links (blog posts `/blog/[slug]`, lab test detail pages `/laborvizsgalatok/[slug]`)
- Does NOT play for: anchor links (`#services`), external links (`https://...`), phone links (`tel:`), email links (`mailto:`), footer utility links
- Simple heuristic: play circle wipe if href starts with `/blog/` or `/laborvizsgalatok/` — everything else navigates normally

**Animation design:**
- When clicked, prevent default navigation
- Mount a fixed full-viewport overlay (`z-[60]` — above intro overlay's z-50)
- Background: pink `#F4DCD6`
- Animate a CSS `clip-path: circle()` from `circle(150% at 50% 50%)` (full coverage) contracting to `circle(12px at 50% 50%)` (small center dot)
- Duration: ~1.0s, ease: `easeInOut`
- After circle wipe completes (~1.0s), hold the pink + dot state for ~0.2s, then execute `router.push(href)`

Wait, re-reading CONTEXT.md: "Circle mask contracts from full viewport to center point". This means:
1. Click link
2. Pink overlay EXPANDS from center (or appears instantly) covering the viewport
3. Then circle contracts to center dot, revealing pink background
4. Then navigate

Actually, simpler interpretation: the entire screen gets a circle-closing wipe effect. The most natural implementation:
1. Click link → mount fixed overlay
2. Overlay starts fully visible as pink
3. A white circle (the page content) shrinks from full size to a center dot via `clip-path: circle()` animating from large to small on the CONTENT behind the overlay. Actually...

Clearest approach per CONTEXT.md:
1. Click triggers overlay mount
2. The overlay has pink background
3. A clip-path circle on the **overlay** expands from center to cover the full viewport (circle grows from 0 to 150%). This creates the "circle wipe" effect — pink expands from center, covering the page content.
4. Final state: full pink screen with a small white dot centered (the dot is a small white circle element)
5. After ~1.2s total: navigate

**Implementation with Motion v12:**
```tsx
<motion.div
  className="fixed inset-0 z-[60] bg-[#F4DCD6]"
  initial={{ clipPath: "circle(0% at 50% 50%)" }}
  animate={{ clipPath: "circle(150% at 50% 50%)" }}
  transition={{ duration: 1.0, ease: "easeInOut" }}
  onAnimationComplete={() => router.push(href)}
>
  {/* Center white dot */}
  <div className="absolute top-1/2 left-1/2 -translate-x-1/2 -translate-y-1/2 w-6 h-6 bg-white rounded-full" />
</motion.div>
```

**State management:**
- Use React state `isWiping` (boolean, default false)
- On click: if href matches internal content pattern, set isWiping=true, which mounts the overlay
- On animation complete: call `router.push(href)` to navigate

**Reduced motion:**
- Import `useReducedMotion` from `motion/react`
- If reduced motion: skip circle wipe, do a quick 0.3s fade overlay instead, then navigate
- Or simply navigate immediately with no animation (simpler, respects preference fully)

**Usage pattern for consumers:**
```tsx
import { CircleWipeLink } from "@/components/motion/CircleWipeLink";

<CircleWipeLink href={`/blog/${post.slug.current}`} className="...">
  {/* card content */}
</CircleWipeLink>
```

The component renders its children inside a clickable element (a `<span>` or `<div>` with `role="link"` and proper keyboard handling, OR a plain `<a>` tag). For SEO, use a real `<a>` tag with the href so crawlers can follow the link. Intercept the click with `onClick` and `e.preventDefault()`.

Actually, the cleanest approach: render a Next.js `<Link>` underneath but intercept with onClick. This preserves prefetching and SEO:
```tsx
<Link href={href} onClick={handleClick} className={className}>
  {children}
</Link>
```

Where `handleClick` checks if circle wipe should play. If yes, `e.preventDefault()` and start animation. If no (non-matching href), let the default Link behavior proceed.

**Portal for overlay:** Use `createPortal` to render the wipe overlay into `document.body` so it's above everything regardless of component nesting. Import from `react-dom`.

**IMPORTANT:** Do NOT create a global state/context for this. Each CircleWipeLink manages its own wipe state independently. Only one link can be clicked at a time in practice.
