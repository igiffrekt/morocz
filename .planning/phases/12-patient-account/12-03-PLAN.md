Create `src/app/api/booking-reschedule/route.ts` with `export const dynamic = "force-dynamic"`:

**POST handler — Atomic Reschedule Swap:**

1. **Parse and validate body** with Zod:
   ```typescript
   const RescheduleSchema = z.object({
     token: z.string().min(1, "Token megadása kötelező."),
     newDate: z.string().regex(/^\d{4}-\d{2}-\d{2}$/, "Érvénytelen dátum."),
     newTime: z.string().regex(/^\d{2}:\d{2}$/, "Érvénytelen időpont."),
   });
   ```

2. **Fetch booking by token:**
   ```typescript
   const booking = await getWriteClient().fetch(
     `*[_type == "booking" && managementToken == $token && status == "confirmed"][0]{
       _id, patientName, patientEmail, service->{name, appointmentDuration},
       slotDate, slotTime, managementToken, "serviceId": service._ref
     }`,
     { token }
   );
   ```
   If not found: return 404.

3. **Enforce 24h window** (same `isWithin24Hours` helper as cancel route):
   If within 24h: return 403 with Hungarian error + clinic phone.

4. **Check the new slot is different from current:**
   If `newDate === booking.slotDate && newTime === booking.slotTime`: return 400 with "Ez a jelenlegi időpont. Kérjük, válasszon másik időpontot."

5. **STEP 1 — Lock the new slot (atomic swap, critical section):**
   ```typescript
   const newSlotId = `${newDate}T${newTime}:00`;
   const newSlotLockDocId = `slotLock-${newDate}-${newTime.replace(":", "-")}`;

   // Create if not exists
   await getWriteClient().createIfNotExists({
     _id: newSlotLockDocId,
     _type: "slotLock",
     slotId: newSlotId,
     status: "available",
   });

   // Fetch for _rev
   const newSlotLock = await getWriteClient().fetch(
     `*[_type == "slotLock" && slotId == $slotId][0]{_id, _rev, status}`,
     { slotId: newSlotId }
   );

   if (!newSlotLock) {
     return Response.json({ error: "Hiba történt. Kérjük, próbálja újra." }, { status: 500 });
   }

   // Check if already booked
   if (newSlotLock.status === "booked") {
     return Response.json(
       { error: "Ez az időpont már foglalt. Kérjük, válasszon másikat." },
       { status: 409 }
     );
   }

   // Optimistic lock on new slot
   try {
     await getWriteClient()
       .patch(newSlotLock._id)
       .ifRevisionId(newSlotLock._rev)
       .set({ status: "booked", bookingRef: { _type: "reference", _ref: booking._id } })
       .commit();
   } catch {
     return Response.json(
       { error: "Ez az időpont már foglalt. Kérjük, válasszon másikat." },
       { status: 409 }
     );
   }
   ```

6. **STEP 2 — Release old slot and update booking (only runs if Step 1 succeeded):**
   ```typescript
   const oldSlotLockDocId = `slotLock-${booking.slotDate}-${booking.slotTime.replace(":", "-")}`;

   // Release old slot
   try {
     await getWriteClient()
       .patch(oldSlotLockDocId)
       .set({ status: "available" })
       .unset(["bookingRef"])
       .commit();
   } catch (err) {
     // Log but don't fail — new slot is already locked, this is accepted degraded state
     console.error("[api/booking-reschedule] Failed to release old slot:", err);
   }

   // Update booking with new date/time
   await getWriteClient()
     .patch(booking._id)
     .set({ slotDate: newDate, slotTime: newTime })
     .commit();
   ```

   **Critical ordering:** Lock new FIRST, release old SECOND. If Step 2 partially fails (old slot not released but booking updated), the degraded state is acceptable — admin can resolve. Log the error: `"[api/booking-reschedule] Partial reschedule: new slot locked but old slot not released."`.

7. **Send reschedule email (fire-and-forget):**
   ```typescript
   if (process.env.RESEND_API_KEY) {
     void sendRescheduleEmailAsync({
       patientName: booking.patientName,
       patientEmail: booking.patientEmail,
       serviceName: booking.service?.name ?? "Szolgáltatás",
       oldDate: booking.slotDate,  // format with toLocaleDateString("hu-HU", ...)
       oldTime: booking.slotTime,
       newDate,                     // format with toLocaleDateString("hu-HU", ...)
       newTime,
       managementToken: booking.managementToken,
     });
   }
   ```
   Use `buildRescheduleEmail` from `booking-email.ts`. Send from `noreply@moroczmedical.hu` with subject `"Időpont áthelyezve — Mórocz Medical"`.

8. **Return 200** with `{ success: true, message: "Az időpont sikeresen áthelyezve." }`.

**Hungarian error messages for all failure cases.** All text with proper accented characters.
