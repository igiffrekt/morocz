  <action>
Create `src/app/api/revalidate/route.ts` as a Next.js App Router route handler.

**HMAC Validation:**
- Read `SANITY_REVALIDATE_SECRET` from `process.env` (not from sanity/env.ts — this is a server-only secret, not a NEXT_PUBLIC_ var)
- Parse the raw request body as text (need raw body for HMAC comparison)
- Extract the `x-sanity-signature` header value (Sanity sends HMAC-SHA256 of the body using the shared secret)
- Compute HMAC-SHA256 of the raw body using `crypto.createHmac('sha256', secret).update(rawBody).digest('hex')`
- Use `crypto.timingSafeEqual` to compare (prevent timing attacks) — both values must be Buffers of equal length
- If no signature header or mismatch → return `NextResponse.json({ message: 'Invalid signature' }, { status: 401 })`
- If `SANITY_REVALIDATE_SECRET` is not set → return 500 with "Webhook secret not configured"

**Revalidation Logic:**
- Parse the raw body as JSON after validation
- Extract `_type` from the Sanity webhook body (Sanity sends `{ _type, _id, ... }` for document changes)
- Map `_type` to revalidation tag(s) using a lookup object:
  ```
  const typeToTags: Record<string, string[]> = {
    homepage: ["homepage"],
    siteSettings: ["siteSettings"],
    service: ["service"],
    serviceCategory: ["serviceCategory"],
    labTest: ["labTest"],
    testimonial: ["testimonial"],
    blogPost: ["blogPost"],
    blogCategory: ["blogCategory"],
    privacyPolicy: ["privacyPolicy"],
  }
  ```
- For each tag in the resolved array, call `revalidateTag(tag)` from `next/cache`
- If `_type` is not in the lookup → still return 200 but log that the type was unrecognized (don't fail — Sanity might send internal document types)
- Return `NextResponse.json({ revalidated: true, tags })` with status 200

**Also export only POST:**
- Do NOT export GET. Only export `POST`.
- No need for a separate GET handler — the webhook only sends POST requests.

**Update .env.example:**
- Add `SANITY_REVALIDATE_SECRET=` line under the existing Sanity vars with comment `# Shared secret for Sanity webhook HMAC validation`
