    **DPIA Document:**
    Create `.planning/dpia/DPIA.md` — a Data Protection Impact Assessment for the booking module. This is internal documentation (not public-facing), stored in the repo per CONTEXT.md decision.

    Structure the DPIA following GDPR Article 35 requirements:
    1. **Description of processing** — What data: patient name, email address, phone number. Why: to book, confirm, and manage medical appointments. How: collected via web form, stored in Sanity CMS (cloud, EU-hosted with DPA), processed by Next.js API routes on Vercel.
    2. **Necessity and proportionality** — Only minimum data collected (name for identification, email for confirmation/reminders, phone for urgent contact). No medical history, no health data (GDPR Article 9 special category). Appointment type is the only health-adjacent data (which service booked).
    3. **Risks to data subjects** — Unauthorized access to booking data, data breach exposing contact details, unintended retention beyond necessary period.
    4. **Mitigation measures** — Sanity DPA signed (data processor agreement), HTTPS-only data transmission, Auth.js session management with JWT, role-based access (patient sees own bookings only, admin sees all), no patient data in client-side localStorage (server-side only), Vercel HTTPS endpoints.
    5. **Data retention** — Booking records retained for 1 year after appointment date for scheduling history, then soft-deleted. Patient account data retained while account is active.
    6. **Data subject rights** — Right to access (patient dashboard shows own data), right to erasure (account deletion removes personal data), right to rectification (patient can update contact details).
    7. **Legal basis** — GDPR Article 6(1)(a) consent (explicit checkbox at registration) and Article 6(1)(b) contract performance (booking is a service contract).
    8. **Third-party processors** — Sanity.io (CMS, DPA required), Vercel (hosting), Resend (transactional email, future Phase 11+).

    All content in English (internal technical document). Include a header with date, assessor ("Claude / Developer"), and status ("Draft — pending doctor review").
