**GET /api/slots/route.ts** — Public endpoint, no auth required (BOOK-07: browsing is pre-auth).

Query params: `date` (required, YYYY-MM-DD), `serviceId` (required, Sanity document ID).

Logic:
1. Validate query params. Return 400 if missing or malformed (Hungarian error message).
2. Fetch in parallel using `sanityFetch` (from `@/sanity/lib/fetch`):
   - `weeklyScheduleQuery` (tag: "weeklySchedule")
   - `blockedDatesQuery` (tag: "blockedDate")
   - `bookingsForDateQuery` with `{date}` param (tag: "booking")
   - `slotLocksForDateQuery` with `{datePrefix: date}` param (tag: "slotLock")
   - Service document by ID: `*[_type == "service" && _id == $serviceId][0]{name, appointmentDuration}` (tag: "service")
3. If service not found, return 404.
4. Extract booked slot times from bookings + active holds from slotLocks.
5. Call `generateAvailableSlots()` from `@/lib/slots` with:
   - schedule: weeklySchedule data
   - blockedDates: array of date strings from blockedDate.dates
   - bookedSlots: times from confirmed bookings
   - heldSlots: times from active holds (status "held" with heldUntil > now — already filtered by GROQ query)
   - date, serviceDurationMinutes from service.appointmentDuration (default 20), maxDaysAhead: 30
6. Return JSON: `{ slots: string[], date: string, serviceName: string, durationMinutes: number }`

Add `export const dynamic = "force-dynamic"` to prevent Next.js static caching of this route.

**POST /api/booking/route.ts** — Authenticated endpoint for booking creation.

Add `export const dynamic = "force-dynamic"` at top.

Request body (validated with Zod):
```typescript
const BookingSchema = z.object({
  serviceId: z.string().min(1, "Kérjük, válasszon szolgáltatást."),
  slotDate: z.string().regex(/^\d{4}-\d{2}-\d{2}$/, "Érvénytelen dátum."),
  slotTime: z.string().regex(/^\d{2}:\d{2}$/, "Érvénytelen időpont."),
  patientName: z.string().min(2, "A név megadása kötelező."),
  patientEmail: z.string().email("Érvénytelen e-mail cím."),
  patientPhone: z.string().min(7, "Kérjük, adja meg telefonszámát."),
});
```

Critical booking sequence:
1. **Auth check:** `auth.api.getSession({ headers: await headers() })` — return 401 with `{ error: "Kérjük, jelentkezzen be." }` if no session.
2. **Validate body** with BookingSchema. Return 400 with field errors if invalid.
3. **Verify slot availability** — re-fetch current bookings for the date to ensure slot is still free. If already booked, skip to step 6 (conflict response).
4. **Get or create slotLock document:**
   - Compute slotId: `{slotDate}T{slotTime}:00`
   - Try to fetch existing slotLock by slotId using `getWriteClient().fetch(slotLockByIdQuery, {slotId})`
   - If no document exists, create one with `createIfNotExists`: `{ _id: "slotLock-{slotDate}-{slotTime}", _type: "slotLock", slotId, status: "available" }`
   - If document exists and status is "booked", go to conflict response (step 6)
   - Re-fetch to get the `_rev` after createIfNotExists
5. **Optimistic lock — patch with ifRevisionID:**
   ```typescript
   try {
     await getWriteClient()
       .patch(slotLock._id)
       .set({ status: "booked" })
       .commit({ ifRevisionID: slotLock._rev });
   } catch (err) {
     // 409: another booking won the race — go to conflict response
   }
   ```
6. **On conflict (409):**
   - Re-fetch available slots for the same date using the same logic as GET /api/slots
   - Pick up to 5 nearest alternative slots (sort by time distance from the requested slot)
   - Return: `{ error: "Ez az időpont sajnos már foglalt. Kérjük, válasszon másik időpontot.", alternatives: ["09:40", "10:00", "10:20"] }` with status 409
7. **Create booking document:**
   ```typescript
   const booking = await getWriteClient().create({
     _type: "booking",
     service: { _type: "reference", _ref: serviceId },
     slotDate,
     slotTime,
     patientName,
     patientEmail,
     patientPhone,
     userId: session.user.id,
     status: "confirmed",
     createdAt: new Date().toISOString(),
   });
   ```
8. **Update slotLock with bookingRef:**
   ```typescript
   await getWriteClient()
     .patch(slotLock._id)
     .set({ bookingRef: { _type: "reference", _ref: booking._id } })
     .commit();
   ```
9. **Send confirmation email (fire-and-forget):**
   - Import getResend pattern (same as admin invite): lazy `new Resend(process.env.RESEND_API_KEY)` inside function
   - Format date using `new Date(slotDate).toLocaleDateString("hu-HU", { year: "numeric", month: "long", day: "numeric", weekday: "long" })`
   - Fetch service name from Sanity for the email
   - Build cancel/reschedule URLs: `/fiokom?cancel={booking._id}` and `/fiokom?reschedule={booking._id}`
   - Use `void getResend().emails.send(...)` with `from: "noreply@moroczmedical.hu"`, subject: `"Időpontfoglalás visszaigazolása — Mórocz Medical"`, html from `buildConfirmationEmail()`
10. **Return success:** `{ bookingId: booking._id, message: "Időpont sikeresen lefoglalva." }` with status 201

Import auth from `@/lib/auth`. Import headers from `next/headers`. Import z from `zod`. Import getWriteClient from `@/lib/sanity-write-client`. Import buildConfirmationEmail from `@/lib/booking-email`.
