**sanity-write-client.ts** — Lazily-initialized Sanity client with write token:
```typescript
import { createClient } from "@sanity/client";

let _writeClient: ReturnType<typeof createClient> | null = null;

export function getWriteClient() {
  if (!_writeClient) {
    _writeClient = createClient({
      projectId: process.env.NEXT_PUBLIC_SANITY_PROJECT_ID!,
      dataset: process.env.NEXT_PUBLIC_SANITY_DATASET!,
      apiVersion: process.env.NEXT_PUBLIC_SANITY_API_VERSION || "2024-01-01",
      token: process.env.SANITY_WRITE_TOKEN,
      useCdn: false,
    });
  }
  return _writeClient;
}
```
Use lazy init pattern (same as db/index.ts and auth.ts) to avoid build-time crashes when SANITY_WRITE_TOKEN is not set. Add biome-ignore comment for the non-null assertions on env vars.

**slots.ts** — Pure slot generation function:

```typescript
export interface SlotGenerationInput {
  schedule: {
    defaultSlotDuration: number;
    bufferMinutes: number;
    days: Array<{
      dayOfWeek: number;
      isDayOff: boolean;
      startTime: string;
      endTime: string;
    }>;
  };
  blockedDates: string[];       // ISO date strings like "2026-03-15"
  bookedSlots: string[];        // Time strings like "09:00" (already booked for the target date)
  heldSlots: string[];          // Time strings like "09:20" (held but not yet booked)
  date: string;                 // Target date "2026-03-15"
  serviceDurationMinutes: number;
  maxDaysAhead: number;         // 30
}

export function generateAvailableSlots(input: SlotGenerationInput): string[]
```

Algorithm:
1. Parse target date. Compute daysAhead from today. If < 0 or > maxDaysAhead, return [].
2. If date is in blockedDates, return [].
3. Find day config by dayOfWeek (JS `getDay()` returns 0=Sun..6=Sat, matches weeklySchedule dayOfWeek values).
4. If no config or isDayOff or missing startTime/endTime, return [].
5. Parse startTime/endTime to minutes. Step through in 20-minute increments (base granularity).
6. For each candidate slot at currentMinutes:
   - Check if serviceDuration fits before endTime: `currentMinutes + serviceDurationMinutes <= endMinutes`
   - For services longer than 20 minutes, check ALL 20-minute sub-slots within the service window are not in bookedSlots or heldSlots
   - If clear, add the slot time string "HH:MM" to results
7. Step increment is 20 minutes (base granularity), NOT 20 + buffer. Buffer is between appointments, so the next available start is: `previous_service_end + bufferMinutes`. However, since we check against bookedSlots which represent already-taken times, the slot generation just needs to check availability at 20-min intervals. The buffer is applied when checking if a nearby booked slot's service window (including buffer) conflicts.

IMPORTANT: Store and return time strings as "HH:MM" (local time, no timezone). This avoids the timezone confusion pitfall documented in RESEARCH.md.

Also export a helper: `export function getAvailableDatesInRange(schedule, blockedDates, startDate: string, endDate: string): string[]` — returns array of date strings that have at least one working hour (not blocked, not dayOff). Used by the calendar to highlight available days.

```typescript
// ─── Bookings for Date ──────────────────────────────────────────────────────
// Revalidation tag: "booking"
export const bookingsForDateQuery = defineQuery(
  `*[_type == "booking" && slotDate == $date && status == "confirmed"]{
  _id,
  slotTime,
  service->{_id, appointmentDuration}
}`
);

// ─── Slot Locks for Date ────────────────────────────────────────────────────
// Revalidation tag: "slotLock"
export const slotLocksForDateQuery = defineQuery(
  `*[_type == "slotLock" && slotId match $datePrefix && (status == "booked" || (status == "held" && heldUntil > now()))]{
  _id,
  _rev,
  slotId,
  status,
  heldUntil
}`
);

// ─── Single Slot Lock ───────────────────────────────────────────────────────
// Revalidation tag: "slotLock"
export const slotLockByIdQuery = defineQuery(
  `*[_type == "slotLock" && slotId == $slotId][0]{
  _id,
  _rev,
  status,
  heldUntil
}`
);

// ─── Services for Booking ───────────────────────────────────────────────────
// Revalidation tag: "service"
export const servicesForBookingQuery = defineQuery(
  `*[_type == "service"] | order(order asc){
  _id,
  name,
  appointmentDuration,
  icon
}`
);
```
