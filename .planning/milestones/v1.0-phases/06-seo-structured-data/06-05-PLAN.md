  <action>
    Systematically audit each component for semantic HTML issues. Apply these corrections:

    **Heading Hierarchy (SEO-01):**
    - Homepage should have exactly ONE `<h1>` — the hero headline. Check that no other section uses `<h1>`.
    - Each homepage section heading (Services, Lab Tests, Testimonials, Blog) should use `<h2>`.
    - If any section currently uses `<h3>` or `<h4>` for its main heading, promote to `<h2>`.
    - Blog post detail pages have their own `<h1>` for the post title — this is correct (different page).
    - Privacy policy page has its own `<h1>` — also correct.

    **Landmark Elements (SEO-01):**
    - Header: Ensure it uses `<header>` element. Ensure the navigation links are wrapped in `<nav aria-label="Fo navigacio">` (Main navigation). Check if the mobile menu nav is labeled distinctly.
    - Footer: Ensure it uses `<footer>` element. Navigation columns should be in `<nav aria-label="Lablec navigacio">` (Footer navigation).
    - Each major section: Ensure wrapped in `<section>` with an `aria-labelledby` pointing to its heading ID, OR `aria-label` if the heading is visually hidden.
    - The `<main>` element is already in layout.tsx — verify it exists.

    **Specific Fixes:**
    - Footer: Update the privacy policy link from `privacyPolicyUrl` prop to hardcoded `/adatkezelesi-tajekoztato` (the URL is now a known route, not a CMS field). Keep the existing `privacyPolicyUrl` prop as fallback: use `privacyPolicyUrl ?? "/adatkezelesi-tajekoztato"` so it works whether or not the CMS field is set. Use Next.js `<Link>` instead of `<a>` for internal navigation.
    - If any `<div>` is being used where a semantic element would be appropriate (e.g., `<article>` for blog cards, `<section>` for content sections), replace it.

    **ARIA (SEO-01):**
    - Ensure all interactive elements have accessible labels.
    - Ensure decorative images have `aria-hidden="true"` or empty alt text.
    - Do NOT add ARIA where native HTML semantics already convey meaning (e.g., don't add role="banner" to `<header>`).

    Keep changes minimal and focused — only fix actual semantic issues, don't refactor component logic or styling.

    All ARIA labels and aria-label values MUST be in Hungarian (SEO-09).
