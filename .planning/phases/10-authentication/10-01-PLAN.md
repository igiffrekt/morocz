  <files>

  <action>
    1. Create `src/lib/auth.ts` — the central Better Auth server instance:
       - Import `betterAuth` from "better-auth"
       - Import `drizzleAdapter` from "better-auth/adapters/drizzle"
       - Import `admin` from "better-auth/plugins"
       - Import `nextCookies` from "better-auth/next-js"
       - Import `db` from "./db"
       - Import `Resend` from "resend" and create instance with `process.env.RESEND_API_KEY`
       - Configure:
         - `database: drizzleAdapter(db, { provider: "pg" })`
         - `emailAndPassword.enabled: true`
         - `emailAndPassword.minPasswordLength: 6` (user decision: low friction)
         - `emailAndPassword.sendResetPassword`: fire-and-forget via Resend (`void resend.emails.send(...)`)
           - from: `noreply@moroczmedical.hu`, subject: `Jelszó visszaállítása`
           - HTML with reset link
         - `emailAndPassword.sendVerificationEmail`: fire-and-forget via Resend
           - from: `noreply@moroczmedical.hu`, subject: `E-mail cím megerősítése`
           - HTML with verification link
         - `socialProviders.google` with `clientId` and `clientSecret` from env
         - `accountLinking.enabled: true`, `accountLinking.trustedProviders: ["google"]`
         - `session.expiresIn: 60 * 60 * 24 * 30` (30 days per user decision)
         - `session.updateAge: 60 * 60 * 24` (refresh daily while active)
         - `rateLimit.window: 60`, `rateLimit.max: 100`
         - `rateLimit.customRules["/sign-in/email"]`: window 60, max 5
         - `plugins: [admin(), nextCookies()]`
       - Export `auth`

    2. Create `src/lib/auth-client.ts` — the React client:
       - Import `createAuthClient` from "better-auth/react"
       - Import `adminClient` from "better-auth/client/plugins"
       - Create and export `authClient = createAuthClient({ baseURL: process.env.NEXT_PUBLIC_APP_URL, plugins: [adminClient()] })`
       - Destructure and export: `signIn, signUp, signOut, useSession`
       - Add `NEXT_PUBLIC_APP_URL=http://localhost:3000` to `.env.example` if not already present

    3. Generate Better Auth database schema:
       - Run `npx @better-auth/cli@latest generate` — this outputs the Drizzle schema for auth tables
       - The output should be placed into `src/lib/db/schema.ts`, replacing the placeholder
       - The schema will contain tables: user, session, account, verification
       - After generation, verify the schema file has proper exports

    4. Run Drizzle migrations:
       - Run `npx drizzle-kit generate` to create migration SQL files
       - Run `npx drizzle-kit migrate` to apply them to the Neon database
       - NOTE: This requires DATABASE_URL to be set. If not available, document the commands in a comment at the top of schema.ts and in the SUMMARY so the user knows to run them.

    5. Create `src/app/api/auth/[...all]/route.ts`:
       - Import `auth` from `@/lib/auth`
       - Import `toNextJsHandler` from "better-auth/next-js"
       - Export `const { GET, POST } = toNextJsHandler(auth)`

    6. Create `middleware.ts` at the project ROOT (NOT src/):
       - Import `NextRequest`, `NextResponse` from "next/server"
       - Import `getSessionCookie` from "better-auth/cookies"
       - In the middleware function:
         - Get session cookie via `getSessionCookie(request)`
         - For `/admin` paths: if no session cookie, let the request through (the /admin page renders its own login form when no session). This is intentional — middleware only does fast cookie check; real role verification happens in the Server Component.
         - For all other paths: pass through
       - Export matcher config that excludes: api, _next/static, _next/image, favicon.ico, studio
       - IMPORTANT: This is Next.js 15 — use middleware.ts, NOT proxy.ts

    IMPORTANT NOTES:
    - Use `void` for email sends (fire-and-forget) to avoid blocking auth responses on Vercel.
    - Do NOT store role only in JWT — Better Auth admin plugin stores it in the database user.role column.
    - Rate limiting returns HTTP 429; the client handles this by showing "Kerjuk, varjon egy percet" (proper Hungarian with accents: "Kerjuk, varjon egy percet" -> "Kérjük, várjon egy percet").
