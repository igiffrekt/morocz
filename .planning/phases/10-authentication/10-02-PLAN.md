  <name>Task 1: Build AuthStep tabbed component with Google OAuth and email/password forms</name>
  <files>

  <action>
    Create `src/components/auth/AuthStep.tsx` — a "use client" component with these specifications:

    **Props:**
    - `onSuccess: () => void` — called after successful auth (booking flow continues)
    - Optional `defaultTab?: "belepes" | "regisztracio"` defaulting to "belepes"

    **State:**
    - `tab`: "belepes" | "regisztracio" (Belépés / Regisztráció toggle)
    - `email`, `password`, `name` (name only for registration)
    - `errors`: record of field-level error messages (inline validation)
    - `loading`: boolean for button disabled state
    - `globalError`: string for server-side errors (e.g., "Hibás e-mail vagy jelszó")
    - `showForgotPassword`: boolean to toggle forgot password sub-view

    **Auto-advance:** Use `useSession()` from auth-client. If session already exists, call `onSuccess()` immediately and return null. This handles the case where a user is already logged in when the booking flow reaches the auth step.

    **Layout (per CONTEXT.md locked decisions):**
    1. Centered card design on simple background (use Tailwind: max-w-md mx-auto p-8 bg-white rounded-2xl shadow-lg)
    2. Tab toggle at top: "Belépés" / "Regisztráció" — active tab has primary color underline, inactive is gray
    3. Google OAuth button (PROMINENT, top position):
       - Full-width button with Google icon (inline SVG)
       - Text: "Folytatás Google-fiókkal" (Continue with Google account)
       - On click: `signIn.social({ provider: "google", callbackURL: "/idopontfoglalas" })`
    4. Divider: horizontal line with "vagy" (or) text centered
    5. Email/password form below:
       - **Belépés tab:** email input, password input, submit button "Belépés"
         - On submit: `signIn.email({ email, password, rememberMe: true, callbackURL: "/idopontfoglalas" })`
         - IMPORTANT: Always pass `rememberMe: true` for 30-day session persistence
       - **Regisztráció tab:** name input, email input, password input, submit button "Regisztráció"
         - On submit: `signUp.email({ email, password, name, callbackURL: "/idopontfoglalas" })`
         - After successful signup, call onSuccess (account works immediately, email verification is nudge-only)
    6. "Elfelejtett jelszó?" link below the login form → sets showForgotPassword to true
    7. Contextual message above the card: "Az időpontfoglalás véglegesítéséhez kérjük, jelentkezzen be vagy hozzon létre egy fiókot." (To finalize your appointment, please log in or create an account.)

    **Inline validation (per CONTEXT.md):**
    - Red text directly below each invalid field
    - Email: check basic format (includes @), show "Érvénytelen e-mail cím"
    - Password: min 6 chars, show "A jelszónak legalább 6 karakter hosszúnak kell lennie"
    - Name (registration only): required, show "A név megadása kötelező"
    - Validate on submit, not on each keystroke (to avoid annoying the user)

    **Error handling:**
    - On signIn/signUp error, parse the error and show Hungarian message:
      - Invalid credentials: "Hibás e-mail cím vagy jelszó"
      - Email already registered: "Ez az e-mail cím már regisztrálva van"
      - Rate limited (429): "Kérjük, várjon egy percet, majd próbálja újra"
      - Generic error: "Hiba történt, kérjük próbálja újra"
    - Show global error above the form fields

    **Successful auth:**
    - After successful signIn.email, signUp.email, or Google OAuth return, call `onSuccess()`
    - For Google OAuth: the user returns to `/idopontfoglalas` after Google consent; the `useSession()` hook will detect the session and auto-advance

    **If showForgotPassword is true:** Render `<ForgotPassword onBack={() => setShowForgotPassword(false)} />` instead of the main auth card.

    **Styling:** Use Tailwind classes consistent with the project's design tokens (see globals.css @theme). Use the project's font (Plus Jakarta Sans) implicitly (it's set at layout level). Inputs should have rounded borders, focus rings, and comfortable padding.

    **Hungarian language:** ALL user-visible text MUST use proper Hungarian accented characters (áéíóöőúüű). Never write unaccented Hungarian.
  </action>
