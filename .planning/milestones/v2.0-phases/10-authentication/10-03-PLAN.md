<task type="auto">
  <name>Task 2: Create admin seed script and admin invite API endpoint</name>
  <files>
    scripts/create-admin.ts
    src/app/api/admin/invite/route.ts
  </files>
  <action>
    1. Create `scripts/create-admin.ts` — a standalone script to create the first admin user:
       - Import `auth` from `../src/lib/auth`
       - Read `ADMIN_EMAIL` and `ADMIN_INITIAL_PASSWORD` from process.env
       - Call `auth.api.createUser({ body: { email: ADMIN_EMAIL, password: ADMIN_INITIAL_PASSWORD, name: "Admin", role: "admin" } })`
       - Log success or error to console
       - Usage instructions in a comment at the top:
         ```
         // Usage: npx tsx scripts/create-admin.ts
         // Requires: DATABASE_URL, BETTER_AUTH_SECRET, ADMIN_EMAIL, ADMIN_INITIAL_PASSWORD in .env.local
         // Run once to create the first admin account
         ```
       - Add validation: if env vars are missing, log clear error and exit

    2. Create `src/app/api/admin/invite/route.ts` — POST endpoint for admin to invite new admin:
       - Import `auth` from `@/lib/auth`
       - Import `headers` from `next/headers`
       - Import `Resend` from `resend`
       - POST handler:
         a. Verify caller is admin: `const session = await auth.api.getSession({ headers: await headers() })` — if no session or role !== "admin", return 403
         b. Parse request body: `{ email, name }` — the new admin's email and name
         c. Validate with Zod v3 (NOT v4): email is valid, name is non-empty
         d. Generate a temporary password: use `crypto.randomUUID().slice(0, 12)` (12-char random string — good enough for a temp password that must be changed)
         e. Create the new admin user: `auth.api.createUser({ body: { email, password: tempPassword, name, role: "admin" } })`
         f. Send invite email via Resend (fire-and-forget with `void`):
            - from: "noreply@moroczmedical.hu"
            - to: the new admin's email
            - subject: "Meghívó — Morocz Medical Admin"
            - HTML body:
              - "Kedves {name},"
              - "Meghívást kapott a Morocz Medical adminisztrátori felületére."
              - "Bejelentkezési adatai:"
              - "E-mail: {email}"
              - "Ideiglenes jelszó: {tempPassword}"
              - "Kérjük, a bejelentkezés után változtassa meg a jelszavát."
              - Link to /admin
         g. Return 201 with `{ success: true, email }` — do NOT return the temp password in the response
         h. On error (e.g., email already exists): return 409 with `{ error: "Ez az e-mail cím már regisztrálva van" }`

       - IMPORTANT: Only authenticated admins can call this endpoint. This is the gatekeeper for the "invite by existing admin" flow.
       - The invite email includes a temp password (simple approach per RESEARCH.md recommendation — no complex invite token system for a 2-admin setup).

    **Hungarian language in emails:** ALL email content with proper accented characters.
  </action>
